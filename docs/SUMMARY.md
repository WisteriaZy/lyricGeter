## 🌏 翻译功能修复（2026-07-03）

### 问题发现

网易云 API 返回的逐字歌词和翻译歌词**时间戳不同步**：

- 主歌词：`[30,1170](30,390,0)我` → 30ms
- 翻译：`[00:00.30]我爱你` → 300ms
- 差异：270ms

### 解决方案

实现**容差匹配**算法，允许 ±500ms 的时间戳误差：

```python
# 找到最接近的翻译（±500ms 范围内）
best_trans = None
min_diff = 500  # 最大容差

for trans_ms, trans_text in trans_list:
    diff = abs(trans_ms - line_start_ms)
    if diff < min_diff:
        min_diff = diff
        best_trans = trans_text
```

### 输出效果

```spl
[00:00.03]我<00:00.42>愛<00:00.81>你[00:01.32]
我爱你
[00:01.32]君<00:01.82>は<00:02.15>気<00:02.48>づ<00:02.80>い<00:02.96>て[00:06.48]
你没有注意到吧
```

**符合 SPL 标准**：
- 主歌词：逐字格式，含时间戳
- 翻译行：紧跟其后，**无时间戳**

### 测试验证

- ✅ 有逐字歌词
- ✅ 有中文翻译
- ✅ 翻译正确合并

## 🐛 SPL 格式修正（2026-07-03）

### 修正内容

1. **逐字时间戳位置**：改为**字之前**（符合 SPL 标准）
   - ❌ 旧格式：`[00:13.86]ヤ[00:14.12]リ[00:14.30]` （时间戳在字之后）
   - ✅ 新格式：`[00:13.86]ヤ[00:14.12]リ[00:14.30]タ` （时间戳在字之前）

2. **行结束时间戳**：显式补充下一行开始时间
   - 符合 SPL "显式行结尾" 语法

3. **时间戳格式**：保持 2 位毫秒（10ms 精度 / 厘秒）
   - 符合 SPL 规范：毫秒部分 1-6 位，通常 2 位

### SPL 官方标准参考

已集成官方文档：`docs/SPL 格式（Salt Player Lyrics）语法标准  不要糖醋放椒盐（椒盐音乐官网）.md`

**关键规则**：
- 时间戳格式：`[mm:ss.xx]`
- 逐字标记：时间戳在文字**之前**
- 行结尾：显式时间戳或隐式（下一行开始）
- 延迟逐字：支持 `<mm:ss.xx>` 包裹中间时间戳

## 🎯 关键突破：网易云逐字歌词

### 发现的格式

网易云 API 返回**专有逐字格式**，不是标准 LRC：

```
[13860,2310](13860,260,0)ヤ(14120,180,0)リ(14300,150,0)タ
```

**格式结构**：
- `[行起始ms,持续ms]` - 行级时间戳
- `(字时间ms,持续ms,0)字` - 每个字的精确时间戳

### SPL 转换结果

```
[00:13.86]ヤ[00:14.12]リ[00:14.30]タ[00:14.45]カ[00:14.67]ネ[00:14.99]ー
```

每个字都有独立的结束时间戳，完美符合 SPL 逐字规范。

## 🔧 技术实现

### 三种格式支持

#### 1. 逐字格式（网易云专有）⭐
- **解析器**：`parser/netease_word.py`
- **转换器**：`_netease_word_to_spl()`
- **覆盖率**：41.7%

#### 2. 混合格式（JSON 元数据 + 标准 LRC）
- 前几行 JSON（作词、作曲）
- 中间标准 LRC（正文歌词）
- 最后 JSON（推广信息）
- **处理**：提取 LRC 行，过滤 JSON

#### 3. 纯 JSON 格式
- **解析器**：`parser/json_lyric.py`
- **格式**：`{"t":时间ms,"c":[{"tx":"文本"}]}`
- **转换器**：`_json_lyric_to_spl()`

### 格式检测优先级

```
converter.py 检测流程：
1. 网易云逐字格式？ → _netease_word_to_spl()
2. 混合格式？       → 提取 LRC + 过滤 JSON
3. 纯 JSON？        → _json_lyric_to_spl()
4. 标准 LRC？       → 直接解析
```

## 🎓 关键经验

### 1. 格式多样性
网易云 API 返回格式因歌曲而异：
- 新歌倾向于逐字格式
- 老歌可能是混合格式
- 部分歌曲纯 JSON 格式

### 2. 正则匹配
逐字格式解析使用两个正则：
```python
LINE_RE = r'^\[(\d+),(\d+)\]'           # 行首
WORD_RE = r'\((\d+),(\d+),\d+\)([^\(]+?)(?=\(|$)'  # 逐字
```

### 3. 时间戳转换
网易云使用毫秒，SPL 使用 `[mm:ss.xx]`：
```python
def _ms_to_stamp(ms: int) -> str:
    m, rem = divmod(ms, 60_000)
    s, ms_part = divmod(rem, 1_000)
    return f"[{m:02d}:{s:02d}.{ms_part // 10:02d}]"
```
